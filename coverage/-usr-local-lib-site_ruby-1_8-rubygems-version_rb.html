8 # The Version class processes string versions into comparable
 
9 # values. A version string should normally be a series of numbers
 
10 # separated by periods. Each part (digits separated by periods) is
 
11 # considered its own number, and these are used for sorting. So for
 
12 # instance, 3.10 sorts higher than 3.2 because ten is greater than
 
13 # two.
 
14 #
 
15 # If any part contains letters (currently only a-z are supported) then
 
16 # that version is considered prerelease. Versions with a prerelease
 
17 # part in the Nth part sort less than versions with N-1 parts. Prerelease
 
18 # parts are sorted alphabetically using the normal Ruby string sorting
 
19 # rules.
 
20 #
 
21 # Prereleases sort between real releases (newest to oldest):
 
22 #
 
23 # 1. 1.0
 
24 # 2. 1.0.b
 
25 # 3. 1.0.a
 
26 # 4. 0.9
